## 🧩 STEP 1: REQUIREMENT DECOMPOSITION (MANDATORY)

Before writing any SQL, break the user's prompt into this exact structured list and write it out:

1. Entities – All tables needed (e.g., `orders`, `customers`, `products`).
2. Required outputs – Exact columns or metrics requested (e.g., `customer_name`, `total_sales`, `rank`).
3. Filters and constraints – Explicit `WHERE` conditions (e.g., `date > '2024-01-01'`).
4. Aggregations – `SUM`, `COUNT`, `AVG`, etc., and their grouping columns.
5. Window functions – `RANK`, `LAG`, `SUM OVER`, etc., with partition and order columns.
6. Time-based logic – Running totals, differences between dates, rolling windows.
7. Group-level constraints – Conditions after aggregation (e.g., `HAVING COUNT(*) >= 3`).
8. Comparative logic – Differences from previous row, rank jumps, percentage changes.

If any category is missing from the prompt, explicitly note "None required."

## 🧠 STEP 2: QUERY DESIGN PLAN

Design the query before writing. Follow these rules:

- Use CTEs for all complex or multi-step logic.
- Separate concerns into sequential CTEs:
  - CTE 1 – Base data with filters and raw columns.
  - CTE 2 – Window calculations (RANK, LAG, running SUM).
  - CTE 3 – Comparisons or differences (e.g., rank jump, sales difference from previous row).
- Never nest window functions inside each other unless your database explicitly supports it (assume no).
- Always specify `ORDER BY` inside `OVER()` for time-based or ranking logic.

## 🛠 STEP 3: SQL GENERATION RULES (ENFORCED)

| Requirement | Correct Pattern | Incorrect Pattern |
|-------------|----------------|--------------------|
| Running total | `SUM(sales) OVER (PARTITION BY region ORDER BY date)` | Missing `ORDER BY` |
| Previous row value | `LAG(sales) OVER (PARTITION BY region ORDER BY date)` | Using subquery instead of LAG |
| Ranking | `RANK() OVER (PARTITION BY dept ORDER BY salary DESC)` | Missing `PARTITION BY` |
| Date difference (current vs previous) | `DATEDIFF(day, LAG(date) OVER (...), date)` | `DATEDIFF(date, date)` (same row) |
| Rank jump | `ABS(rank - LAG(rank) OVER (...))` | Manual calculation without LAG |

NEVER output a query with:
- Missing `ORDER BY` inside a window function that requires it.
- `DATEDIFF` with identical columns from the same row.
- Nested window functions (`RANK() OVER (ORDER BY SUM(...) OVER (...))`).
- Ambiguous column aliases.
